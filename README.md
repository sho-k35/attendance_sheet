# 塾向け 出席・成績一括管理システム

学習塾における生徒の出欠状況や小テストの点数、宿題の提出状況を一元管理するためのWebアプリケーションです。
紙や煩雑なエクセル管理からの脱却を想定し、直感的なUIと堅牢なデータ処理を意識して開発しました。

## アプリケーションの目的と特徴
* **直感的な一括操作**: 学年・クラス・学校での柔軟な絞り込み検索と、全員分のデータをワンクリックで保存できる一括保存機能を実装。
* **動的なUI切り替え**: 記録カテゴリ（出欠・小テストなど）に応じて、入力インターフェース（〇×ボタン / 数値入力）がシームレスに切り替わります。
* **導入の容易さ**: Dockerを利用し、コマンド一発でローカル環境にシステムを構築可能です。

## 使用技術・環境
* **Backend**: Python 3.9 / FastAPI
* **Data Processing**: pandas / pydantic
* **Frontend**: HTML5 / CSS3 / Vanilla JavaScript
* **Infrastructure**: Docker / Docker Compose

## 技術的な工夫点

### 1. データの正規化とPandasによる高速処理
運用時のデータ肥大化や拡張性を考慮し、横持ちのクロス集計表ではなく、「生徒マスタ（基本情報）」と「トランザクション（日々の記録）」にデータを分割（縦持ち構造）して設計しました。バックエンドでの結合・抽出処理には `pandas` を採用し、簡潔かつ高速なデータ操作を実現しています。

### 2. Pydanticによる堅牢なバリデーション
FastAPIの `Pydantic` モデルを活用し、フロントエンドから送信されるデータの型検証を厳密に行っています。不正なデータ（例：必須項目の欠落）が送信された場合は `422 Unprocessable Entity` で弾き、システムやCSVファイルの破損を防ぐ防御的プログラミングを実践しています。

### 3. 一括保存時のファイル書き込み競合（排他制御）の回避
フロントエンドで「全員分の一括保存」を行う際、非同期処理による並列リクエスト（`Promise.all`等）を意図的に避け、`for...of` と `await` を用いた**直列通信**を実装しました。これにより、バックエンド側でのCSVファイルの同時書き込みによるロックやデータ破損のリスクをフロントエンド側からコントロールしています。

### 4. シングルオリジン構成と静的ファイルのホスティング
CORSエラーの回避と本番デプロイ時の構成を想定し、FastAPIの `StaticFiles` マウント機能を利用。APIサーバーとフロントエンドのWebサーバーを同一ポートで兼任させる構成にしています。

## ディレクトリ構成

```text
.
├── main.py                 # FastAPIアプリケーション本体
├── requirements.txt        # Python依存ライブラリ
├── Dockerfile              # Dockerビルド設定
├── docker-compose.yml      # コンテナオーケストレーション
├── data/
│   ├── students.csv        # 生徒マスタ（初期データ）
│   └── records.csv         # 日々の記録トランザクション
└── static/                 # フロントエンド静的ファイル
    ├── index.html
    ├── style.css
    └── js/
```

## 起動方法
本アプリケーションは Docker を使用して起動します。事前に Docker Desktop 等のインストールをお願いいたします。

### 1. Docker Compose を使用してコンテナをビルド・起動
```bash
docker-compose up --build -d
```
### 2. ブラウザでアクセスしてシステムを利用
- アプリケーション画面: `http://localhost:8000`
- API ドキュメント: `http://localhost:8000/docs`
### 3. システムの終了
終了する場合は以下のコマンドを実行
```bash
docker-compose down
```
